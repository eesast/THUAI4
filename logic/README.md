# THUAI4 游戏逻辑
THUAI4 原电子系第22届队式程序设计大赛

## 设计思路
目标框架：.NET Core 3.1

#### 游戏建模

- 合理地抽象与组织整个游戏逻辑
- 基于事件驱动实时更新游戏信息
- 写回放文件（增量更新）
- 注意与通信、玩家交互的交流合作
- （饼）提供合适的debug接口，比如可以手动调整游戏场景

#### 重点

- Protobuf的使用
- 事件驱动的程序设计
- 资源的互斥访问（避免不同线程同时使用数据）
- 物理引擎与游戏主逻辑解耦

#### 难点（饼）

- 设计良好的debug接口
